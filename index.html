Before you start

Prerequisites

Confirm these items before opening Copilot Studio.

Copilot Studio environment

Use a Developer or Sandbox environment. Avoid the default environment if DLP policies block custom connectors or MCP tools.

⚠️ Enterprise MCP setup (critical pre-requisite)

You must complete the pre-requisite steps before the workshop. The Enterprise MCP server requires an Entra app registration, API permissions with admin consent, and PowerShell provisioning that cannot be done live in a few minutes.

Follow both guides in order:

  1. Get started with the Microsoft MCP Server for Enterprise — PowerShell provisioning and app registration.
  2. Use Microsoft MCP Server for Enterprise from Copilot Studio — OAuth connector and federated credential setup.

Admin and app registration

You need Cloud Application Administrator or equivalent access, a confidential client app registration, and the required MCP API permissions consented.

Work IQ Copilot

Confirm Microsoft 365 Copilot licensing and that the Work IQ Copilot MCP server is allowed for your tenant.

Optional runbooks

Have a SharePoint site with rollout checklists, service-owner notes, or change-management runbooks if you want internal citations.

Test identities

Use at least two users with different roles and different recent mail/calendar/Teams activity to show permission-bounded answers.

Hands-on build

Create and configure the agent

  1. Create the agent shell

    1. Go to copilotstudio.microsoft.com.
    2. Select your Developer or Sandbox environment.
    3. Select Create > New agent.
    4. Enter the agent name Roadmap Concierge, then select Create.
    5. After the agent loads, set the description and model.
    DescriptionYour IT-pro briefing agent - fuses Microsoft Roadmap and Azure Updates with live tenant data and Work IQ context.
    ModelGPT-5.5 Instant

    GPT-5.5 Instant is experimental and might not be enabled by your administrator. If it is unavailable, choose the closest GPT-5.5 model your environment allows.

  2. Add the agent instructions

    Paste this into the Instructions field.

    You are Roadmap Concierge, an IT-pro agent for Microsoft 365 and Azure
change readiness. You run on GPT-5.5 Instant, so optimize for fast,
concise, grounded answers.

Audience: Microsoft 365 admins, change managers, service owners, and
engineering leads. Assume technical fluency. Do not explain basic admin
concepts unless asked.

CORE BEHAVIOR

1. Answer with evidence
   - Use available tools and knowledge whenever the user asks about
     current releases, tenant state, internal content, or recent work
     context.
   - Do not answer from general knowledge when a grounded source is
     needed.
   - If a lookup returns no results, say that clearly. Do not invent
     Roadmap IDs, dates, tenant counts, policies, owners, or decisions.

2. Keep source boundaries clear
   - Separate public release information, live tenant facts, internal
     knowledge, and personal/work-context signals.
   - Treat tenant data as authoritative for tenant-readiness questions.
   - Treat work-context signals as helpful context, not tenant-wide
     inventory or compliance evidence.
   - When sources disagree or are incomplete, explain the gap instead
     of forcing a conclusion.

3. Synthesize readiness
   - For "are we ready", "what should we do", or "who needs to know"
     questions, combine the relevant evidence into:
       * What is changing
       * Who or what appears affected
       * Current readiness signals
       * Risks, blockers, or unknowns
       * Recommended next actions
   - Keep recommendations practical and specific.

4. Cite and attribute
   - Include Microsoft 365 Roadmap IDs or Azure Updates IDs when
     discussing specific release items.
   - Cite internal knowledge sources when available.
   - Label work-context findings as recent mail, meeting, or Teams
     context when used.

5. Draft useful communications
   - When asked to draft a leadership update, IT bulletin, Teams post,
     or change notice, produce a complete send-ready draft.
   - Ground the draft in retrieved evidence. Include dates, IDs,
     affected groups, owners, and open risks when known.
   - Avoid placeholder language unless the underlying fact is unknown;
     in that case, mark it as a decision or data gap.

6. Handle permissions and safety
   - Respect the signed-in user's permissions and roles.
   - Do not reveal tenant, mailbox, meeting, Teams, or document content
     beyond what the user is allowed to access.
   - Summarize sensitive work-context content instead of quoting it
     unless a quote is necessary and appropriate.
   - If asked to make a change and no approved write capability is
     available, say you cannot make the change directly and offer a
     draft request, checklist, or implementation plan.

STYLE

- Lead with the answer, then the key evidence.
- Be direct, technical, and concise.
- Use short bullets or a compact table when comparing items.
- Ask a clarifying question only when the missing detail changes which
  source to use or what action to recommend.
- Do not describe your tools or internal process.

  3. Add the Microsoft Release Communications MCP server

    1. Go to Tools > Add a tool > Model Context Protocol.
    2. Use these values.
    Server nameMRC-MCP-Server
    Server URLhttps://www.microsoft.com/releasecommunications/mcp
    AuthenticationNone

    This public MCP server adds Microsoft 365 Roadmap and Azure Updates tools.

  4. Add the Microsoft MCP Server for Enterprise

    This server provides read-only Microsoft Graph access through the signed-in user's permissions.

    ⚠️ Pre-requisite: You must have already completed the Get started guide (PowerShell provisioning + app registration) before proceeding. If you haven't, stop here and complete it first.

    1. In Copilot Studio, add a new MCP tool.
    2. Set the server URL to https://mcp.svc.cloud.microsoft/enterprise.
    3. Choose OAuth 2.0 > Manual.
    4. Copy the redirect URL from Copilot Studio and add it to your Entra app registration as a Web redirect URI.
    5. In Power Apps, edit the custom connector security settings and enable service-principal support with managed identity.
    6. Create the matching federated credential on the app registration.
    7. Create and test the connection, then return to Copilot Studio and select Add and configure.

    If the connector test fails after adding federated credentials, verify the issuer, subject, and credential name match the connector values exactly.

    Full walkthrough: Use Microsoft MCP Server for Enterprise from Copilot Studio

  5. Add Work IQ Copilot

    1. Go to Tools > Add a tool > Model Context Protocol.
    2. Search for Work IQ Copilot.
    3. Select the Work IQ Copilot MCP server.
    4. Create a new connection and sign in with your demo user.
    5. Run one work-context prompt in the test pane and select Allow if prompted.

    Seed the demo user with relevant emails, Teams posts, and meetings before testing so Work IQ Copilot has realistic context.

  6. Optional: add SharePoint runbooks

    1. Go to Knowledge > Add knowledge > SharePoint.
    2. Paste the runbook library URL, such as https://<tenant>.sharepoint.com/sites/itops/runbooks.
    3. Wait for indexing and enable citations.

  7. Keep topics minimal

    Leave only the system Greeting topic enabled. Let generative orchestration choose the right tools and knowledge sources.

Validation

Test the agent in Copilot Studio

Run these prompts in the test pane. Watch the Activity map to confirm the orchestrator selects the right source.

Roadmap only

What Microsoft Teams features are releasing in the next 30 days?

Expect Roadmap items, release timing, and IDs.
Specific item

What's the status of Roadmap feature ID 526798?

Expect one detailed item or a clear not-found answer.
Tenant facts

How many users in my tenant have a Copilot license?

Expect Enterprise MCP and live Graph counts.
Work context

What recent emails or Teams conversations mention Teams rollout blockers?

Expect Work IQ Copilot and permission-bounded context.
Joined reasoning

Are any of those upcoming Teams features in preview already inside our tenant, and have people been discussing them recently?

Expect release data, tenant facts, and work-context signals separated.
Retirements

Which Azure services are retiring this quarter? Anything we depend on?

Expect Azure Updates retirement data and optional runbook citations.
Stakeholders

Who should be included in a readiness follow-up based on recent mail, meetings, and Teams context?

Expect suggested stakeholders with reasoning from accessible context.
Communication draft

Draft a leadership update covering the top three Teams changes and any internal concerns from recent work context.

Expect a send-ready update grounded in retrieved evidence.
Use and publish

Publish the agent and use it with your IT team

  1. Open Channels and turn on Microsoft 365 Copilot.
  2. Turn on Microsoft Teams if the agent should be available there.
  3. Submit for tenant admin approval.
  4. After approval, install the agent from Copilot Chat > Agents or from Teams.

What good looks like

  • At least four orchestration paths: Roadmap-only, tenant-only, Work IQ-only, and joined.
  • Roadmap and Azure Update IDs appear when specific release items are discussed.
  • Work IQ findings are labeled as work-context signals, not tenant-wide facts.
  • Internal runbook citations appear when SharePoint knowledge is connected.
  • Drafted communications are complete enough to send after human review.
Troubleshooting

Common issues

Custom connector creation is blocked

Use a Developer or Sandbox environment. The default environment often has DLP policies that block custom connectors.

Graph queries return 403 Forbidden

Confirm the signed-in user has the required Entra role. If role changes were recent, delete and recreate the connection.

Federated credential connection fails

Verify issuer, subject, and credential name exactly match the values shown by the managed identity connector setup.

Roadmap or Azure Updates tools are not selected

Make the user prompt explicit: mention Microsoft 365 Roadmap, Azure Updates, GA dates, previews, or retirements.

Work IQ Copilot returns little context

Make sure the demo user has relevant and accessible email, meeting, and Teams activity. Work IQ Copilot is permission-bound.
